来自jarvis里面的re题

​ 解答人数有点多。我就去凑凑热闹看看能不能水点分。

题目传送门

点开rar文件，然后解压。要求输入注册码。

看了一下，想都不想直接逆向。

拉入ida看，发现一堆图标形式的代码。

头秃，按了F5没有用。

大概出现了图片里面的东西。也不是汇编代码，也不是c或c++。无力看懂

然后往下看了看，还是有部分英语单词认识的。大意就是比对你的输入和注册码。 比对成功就是注册成功了。

那我们就去找找注册码。

第一反应就是那一大段的十六进制的。在线工具转了一下，发现后后面的字符串意识相同。果断pass掉

然后再往下面找。

看到了unicode。这个认识，肯定和注册码逃不了关系。然后看了一下，前后有注册的信息。

那么就是这个，是我们需要的信息。大概可能进行了base64的加密。

看字符串后面的等号。

UENURntFYTV5X0RvX05ldF9DcjRjazNyfQ==

在线工具解了一下。碰碰运气。

PCTF{Ea5y_Do_Net_Cr4ck3r}

How to get my first wife!

​ 总算入门博客了。在 刘大佬赞神的指导光辉照耀下。搭了第一个博客。真的是酷哇。期间膜了曹神的博客。发现曹神的右下角有一个小老婆吕孩。

​ 作为一个死宅 ！！！！

​ 我也应该有对象小姑娘给我看博客大门的。

​ 所以搜教程，顺便搜一下对象。收旧对象，破对象，不要的对象

​ 我，找到小老婆了！！

​ first of all （炫耀一下英语）

​ 先下载一个插件。

​ 小老婆是 Live 2d 模型。百度 hexo +live 2d 都能搜到教程哇。

​ 我是为了写博客练练手先假装自己是一个大佬，写下如何拥有自己的小老婆

​ 首先，安装 npm 包

打开 git bash 输入

npm install --save hexo-helper-live2d

​ 然后在主自己hexo 配置文件 _config.yml中添加如下配置

​ live2d:
​ enable: true
scriptFrom: local
pluginRootPath: live2dw/
pluginJsPath: lib/
pluginModelPath: assets/
tagMode: false
debug: false
model:
​ use: live2d-widget-model-shizuku //这个表示你用哪个模型哇，但是我们现在本地文件里面暂时只有live2d的插件，一个模型都没有，还需要再次下载
display:
​ position: right
​ width: 150
​ height: 300
mobile:

然后，就要下载我们可爱的看门小老婆拉。

live2d模型

点击上面可以跳转到开发这个模型的大佬的博客哇。我觉得它超级厉害。

然后我们确定自己喜欢的模型样式就可以下载拉。

下载方式

npm install live2d-widget-model-XXXXX    *XXXX就是模型的名字啦*
